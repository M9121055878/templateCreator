# پیاده‌سازی هدر در Layout اصلی سایت

## ✅ تغییرات انجام شده

### 1. ایجاد Context برای مدیریت اطلاعات هدر
- **فایل**: `src/contexts/page-header-context.jsx`
- **عملکرد**: مدیریت عنوان، زیرعنوان، دکمه اکشن و دکمه بازگشت

### 2. کامپوننت نمایش هدر در Layout
- **فایل**: `src/layouts/dashboard/header-content.jsx`
- **عملکرد**: نمایش اطلاعات هدر در centerArea هدر اصلی سایت

### 3. به‌روزرسانی Layout داشبورد
- **فایل**: `src/layouts/dashboard/layout.jsx`
- **تغییر**: اضافه کردن `centerArea: <HeaderContent />` به headerSlots

### 4. اضافه کردن Provider به Layout اصلی
- **فایل**: `src/app/dashboard/layout.jsx`
- **تغییر**: پیچیدن children در `PageHeaderProvider`

### 5. تغییر کامپوننت PageHeader
- **فایل**: `src/components/page-header/page-header.jsx`
- **تغییر**: حالا فقط اطلاعات را به Context ارسال می‌کند و چیزی رندر نمی‌کند

### 6. به‌روزرسانی تمام View ها
تمام فایل‌های view به‌روزرسانی شدند تا:
- PageHeader را در ابتدای کامپوننت قرار دهند
- محتوای اضافی Stack و spacing را حذف کنند

## 🎯 نتیجه

حالا عنوان‌ها و دکمه‌ها در هدر اصلی سایت نمایش داده می‌شوند:

```
┌─────────────────────────────────────────────────────────────┐
│ [Menu] [Logo]    [← عنوان صفحه]    [دکمه اکشن] [Account] │
│                   زیرعنوان                                  │
└─────────────────────────────────────────────────────────────┘
```

### ویژگی‌های پیاده‌سازی شده:

✅ **عنوان صفحه در هدر اصلی**: در centerArea هدر نمایش داده می‌شود  
✅ **دکمه اکشن در هدر**: در سمت راست هدر قرار دارد  
✅ **دکمه بازگشت**: برای صفحات جزئیات  
✅ **ریسپانسیو**: در موبایل دکمه‌ها به آیکون تبدیل می‌شوند  
✅ **RTL Support**: پشتیبانی کامل از راست به چپ  

### صفحات پیاده‌سازی شده:

| صفحه | عنوان | دکمه اکشن |
|------|--------|-----------|
| داشبورد | داشبورد | خروج از حساب کاربری |
| عکس ساز | عکس ساز | ذخیره تصویر |
| تم ساز | تم ساز | ساخت تم جدید |
| کاربران | کاربران | کاربر جدید |
| نقش ها | نقش ها | نقش جدید |
| شرکت ها | شرکت ها | شرکت جدید |
| پروفایل | پروفایل | خروج از حساب کاربری |
| جزئیات شرکت | جزئیات شرکت: [نام] | بازگشت به شرکت ها |

## 🔧 نحوه استفاده

در هر view کافی است PageHeader را در ابتدای کامپوننت قرار دهید:

```jsx
export function MyView() {
  return (
    <Container maxWidth="xl">
      <PageHeader
        title="عنوان صفحه"
        subtitle="توضیحات اختیاری"
        action={{
          label: 'دکمه اکشن',
          icon: 'mingcute:add-line',
          onClick: handleAction,
        }}
        backAction={{
          onClick: handleBack, // اختیاری
        }}
      />
      
      {/* محتوای صفحه */}
    </Container>
  );
}
```
